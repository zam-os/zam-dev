# Community Goals

This directory holds the community's shared objectives — the things the group
wants to achieve, collectively owned and maintained.

## What a goal is

A goal describes something the community wants to be able to do, produce, or
become. Goals range from concrete deliverables to long-term capability building.

Goals decompose into tasks (actionable work items) which in turn surface learning
tokens — the concepts members need to understand to execute them well.

## Goal files

Each goal is a markdown file with YAML frontmatter:

```markdown
---
title: Ship zam-core v1.0
status: active           # active | completed | paused | cancelled
created: 2026-03-28
updated: 2026-03-28
---

Why this goal matters and what success looks like.

## Tasks

- [ ] Publish to npm as zam-core
- [ ] Write onboarding documentation
- [ ] Set up CI/CD pipeline
```

## Levels of involvement

Goals may be owned by the community or contributed to externally:
- **Own** — the community is responsible for making it happen
- **Contribute** — the community invests partial effort
- **Vote** — the community endorses but does not actively work on it
