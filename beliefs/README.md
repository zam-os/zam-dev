# Community Beliefs

This directory holds the community's shared worldview — the premises the group
holds to be true. Beliefs guide how the ZAM agent interprets goals and tasks in
the context of this community.

## What a belief is

A belief is a declarative statement the community stands behind. It is not a goal
(something to achieve) or a rule (something to follow) — it is a premise that
shapes how the community thinks and acts.

Examples:
- "Learning is most effective when it happens during real work, not in isolation."
- "Code is owned collectively. Any member can improve any part."
- "Automation should free humans for deeper work, not replace their judgment."

## How beliefs work

Beliefs are markdown files in this directory, each making a specific claim the
community endorses. The ZAM agent reads them as context when assisting community
members with tasks.

Every committed change to `beliefs/` represents a conscious decision — the git
history is the community's intellectual audit trail.

## Guidelines

- Keep each belief file to one core claim (plus supporting reasoning)
- Aim for fewer than seven active beliefs (Miller's number)
- Retire beliefs that no longer hold by archiving them, not deleting them
